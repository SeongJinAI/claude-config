# /project-chat 명령어

프로젝트의 기능명세서, 사용자메뉴얼, 도메인 문서를 기반으로 질문에 답변합니다.

## 사용법

```bash
/project-chat 주문 취소는 어떻게 작동하나요?
/project-chat --source=specs 결제 실패 시 처리 흐름
/project-chat --verbose 사용자 인증 방식
```

## 수행 작업

### 1. 문서 검색

문서 디렉토리에서 관련 문서 탐색:
```
docs/
├── specs/       # 기능명세서 - 개발자 관점
├── manuals/     # 사용자메뉴얼 - 사용자 관점
├── domain/      # 도메인 지식 - 비즈니스 규칙
└── feedback/    # 피드백 기록 - 이슈 히스토리
```

### 2. 관련 문서 식별

질문 키워드 분석 후 관련 문서 우선순위 결정:
1. YAML 프론트매터의 `keywords`, `tags` 매칭
2. 문서 제목 매칭
3. 본문 내용 검색

### 3. 답변 생성

출력 형식:
```
📖 프로젝트 문서 기반 답변
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[답변 내용]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📚 참조 문서:
  - docs/specs/order.spec.md (라인 45-67)
  - docs/domain/payment-rules.md

⚠️  문서에 없는 내용:
  - 해외 결제 관련 정보는 문서에 없습니다
```

## 옵션 설명

| 옵션 | 설명 |
|-----|------|
| `--source=<type>` | 특정 문서 유형만 참조 (specs, manuals, domain, feedback) |
| `--verbose` | 참조 문서 상세 내용 포함 |
| `--suggest` | 문서 개선 제안 포함 |

## 답변 원칙

### 문서에 있는 내용
- 정확한 출처와 함께 답변
- 관련 문서 라인 번호 명시

### 문서에 없는 내용
- 명확히 "문서에 없음" 표시
- 코드에서 유추 가능한 경우 별도 표시
- 문서 보완 필요 여부 안내

### 모호한 경우
- 여러 문서에서 상충되는 정보가 있으면 모두 언급
- 확인이 필요한 부분 명시

## Phase 확장 계획

**Phase 1 (현재)**: 내부용
- 개발자/기획자가 프로젝트 이해를 위해 사용
- CLI 기반 인터페이스

**Phase 2 (향후)**: 외부용
- 실제 사용자가 제품 문의에 사용
- MCP 서버 연동으로 웹 인터페이스 제공
- 사용자 질문 로깅 및 FAQ 자동 생성

## 연계 명령어

- `/verify-docs` - 문서-코드 동기화 확인
- `/gen-spec` - 누락된 문서 생성

**IMPORTANT**: 항상 답변의 출처를 명시하고, 문서에 없는 내용은 반드시 별도로 표시하세요.
